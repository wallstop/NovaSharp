# Lua Compatibility (Target: 5.4.8)

NovaSharp aims to match the behaviour of upstream Lua **5.4.8**.

See `docs/compatibility/compatibility-profiles.md` for the detailed per-version feature flags backing the `LuaCompatibilityVersion` API.

## Compatibility Principles

- Align runtime semantics, standard library behaviour, and coroutine semantics with stock Lua 5.4.8.
- Prefer behaviour parity over historic legacy quirks; regressions against Lua take precedence.
- Track any intentional deviations (e.g., .NET integration helpers) in this document and mirror coverage tests where feasible.

Use the matrices below to understand how close NovaSharp is to stock Lua 5.4.8, which gaps still need work, and which tests guard each area. Update the tables when functionality lands or coverage moves.

## Compatibility Modes

NovaSharp now exposes a version selector so applications can pin script execution to a specific Lua baseline. The global entry point is `Script.GlobalOptions.CompatibilityVersion`, and per-script overrides live on `Script.Options.CompatibilityVersion`. Both default to `LuaCompatibilityVersion.Latest`, keeping behaviour aligned with the most recent NovaSharp target (Lua 5.4.x today). Future releases can introduce compatibility shims for Lua 5.5/5.3/5.2 features without forcing a wholesale migration‚Äîset the enum to the desired version before running scripts to opt in.
